工单流转平台 - 系统架构

告警接收 → 工单创建 → 客户推送 → 授权执行 → 完成归档

核心服务层 (Backend)

  • • Webhook Receiver: 接收告警，HMAC-SHA256 签名校验
  • • Alert Parser: 解析 title/severity/fingerprint 等字段
  • • Ticket Service: 工单创建、30min 去重窗口、告警追加
  • • Worker Pool: 异步推送队列 (⚠️ 待接入实际推送)

数据层 (Data)

  • • PostgreSQL: 工单、客户、告警源、审计日志持久化
  • • Redis: 去重缓存 (fingerprint → ticket_id 映射)
  • • 数据库迁移: Alembic 管理，支持回滚

安全机制 (Security)

  • • JWT Token: 管理员/操作员 RBAC 权限控制
  • • HMAC-SHA256: 告警源 Webhook 签名验证
  • • API Key + Secret: 客户系统对接认证
